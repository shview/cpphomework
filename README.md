# Last Command - Modded (贪吃蛇 x 弹幕射击 融合机制游戏)

本项目是一个基于 C++ 和 SFML 3.x 开发的 2D 动作弹幕游戏大作业。游戏灵感及核心世界观致敬了《Last Command》（最后指令）。
本作创新性地将“贪吃蛇（Snake）”的体长机制与“弹幕躲避（Bullet Hell）”和“Boss战”结合，带来了高风险高收益的“解析”战斗系统。

---

## 🎮 核心玩法概述 (Core Gameplay)

玩家控制一个由代码数据组成的“贪吃蛇”游走于赛博空间中。
- **躲避与变长**：地图上会不断刷新“数据点”，玩家拾取后身体会变长。体长越长，受击判定风险越大，但能提供的攻击力与解析速度也越高。
- **解析与攻击**：玩家没有常规的主动开火键。必须在拥有至少 3 节身体时，按住 `D` 键进入“解析模式”（此时移速骤降），解析进度条满后才能向 Boss 发射出致命的追踪伤害点。
- **战损与掉落**：玩家受到伤害时，会失去所有积攒的额外身体数据，但这些数据会以 60% 的比例散落在周围，玩家可以冒着弹幕的风险重新夺回它们。

## ⌨️ 操作说明 (Controls)

| 按键 | 动作 | 详细说明 |
| :--- | :--- | :--- |
| `↑` `↓` `←` `→` | 移动 | 控制蛇头朝四个绝对方向移动。 |
| `Space` (长按) | 聚焦减速 | 舍弃移动速度以换取走位精度，移速降至 69.1。 |
| `D` (长按) | 解析模式 | 进入解析状态，进度满 100% 后自动对 Boss 发起攻击。此状态下移速等同于 `Space` 减速。 |
| `LeftShift` / `F` | 冲刺 (Dash) | 朝当前移动方向快速冲刺一段距离。冲刺期间及之后 0.3 秒内处于**无敌状态**。 |
| `Q` | 释放冲击波 | 消耗 35 点能量，在蛇头位置释放一个不断扩大的冲击波，能**消除触碰到的所有敌人弹幕**。 |
| `F1` | 作弊代码 | 瞬间秒杀当前所有 Boss（用于测试）。 |
| `Esc` | 暂停/返回 | 在游戏中呼出暂停菜单，在菜单中返回上一级。 |

## ⚙️ 游戏系统详细数值设定 (Numerical Systems)

### 1. 玩家 (Player/Snake)
- **生命值 (Health)**：初始 3 点，上限 5 点。
- **能量值 (Energy)**：上限 100 点，自动恢复速度 `12 点/秒`。释放一次冲击波消耗 `35` 点。
- **冲刺 (Dash)**：充能上限 2 次，每消耗一次需要 `2.5秒` 重新充能。冲刺持续时间 `0.15秒`，无敌帧持续 `0.4秒`。
- **移动速度 (Speed)**：
  - 基础移速：`199.7 px/s`
  - 减速/解析移速：`69.1 px/s`
  - 冲刺移速：`1075.2 px/s`
- **体长与解析 (Length & Parse)**：
  - 基础体长：2 节（最大 10 节）。节与节之间的跟随间距 `gap = 5`。
  - 解析门槛：需要体长 `>= 3` 才能按 `D` 解析。
  - 解析速度：基础 `30/s`，每多一节额外提供 `+15/s`（即满长时 1 秒多即可解析完毕）。若松开 `D`，进度以 `-50/s` 衰减。
  - 攻击伤害：`10 + 当前体长 * 8`。

### 2. 拾取物与掉落机制 (Items & Drops)
地图上存在 3 种数据点：
- 🔵 **普通数据 (Cyan)**：每拾取 1 个，体长 +1。场上无数据时，每 `1.2秒` 在 Boss 周围安全区随机生成一个。
- 🟢 **恢复数据 (Green)**：场上唯一存在。如果场上没有，则每 `10秒` 生成一个。拾取后 HP +1（上限 5），并重置 10 秒倒计时。
- 🟠 **散落数据 (Orange)**：玩家受到攻击时，会扣除 1 滴血并将体长强制重置为 2。损失的体长会产生 `ceil((当前体长 - 2) * 0.6)` 个碎片，散落在玩家周围 `40~100` 像素范围内。碎片拥有外发光特效，存活 `10秒`，随时间逐渐透明直到消失。拾取可恢复体长。

### 3. Boss 与难度设置 (Bosses & Difficulty)
游戏包含 3 种难度（Easy, Normal, Hard），直接影响 Boss 的**基础血量**与**弹幕发射频率**。
- **基础血量 (单阶段)**：Easy `35` / Normal `75` / Hard `125`
- **基础弹幕间隔**：Easy `0.36s` / Normal `0.24s` / Hard `0.16s`
- **阶段机制 (Phases)**：每个 Boss 拥有 3 个血条阶段（Phase 0, 1, 2），每个阶段对应不同的弹幕发射模式。如果处于双 Boss 关卡，击破其中一个 Boss 的当前阶段后，它将进入 `PhaseWait`（隐身等待）状态，直到另一个 Boss 也被击破当前阶段，双方才会同时复活并进入下一阶段。

### 4. Boss 攻击模式详解 (Attack Patterns)
Boss 拥有一个核心判定点（Hitbox 半径 20px）。它的攻击分为：
- **普通弹幕 (Normal)**：伤害 1。碰撞半径 12px。根据所处阶段，发射旋转的十字弹幕或双向风车弹幕。
- **天降特攻 (Special)**：伤害 1。碰撞半径 35px。每 `10~12秒` 触发，会在玩家当前蛇头、蛇头行进预测点（前方 200px）、蛇尾预测点（后方 150px）、以及蛇身中点，生成红线预警，`1.5秒` 后落下巨大光柱或巨型子弹。预警线会“死死咬住”并跟随 Boss 本体移动。
- **极速泡泡 (Bubble)**：伤害 1。碰撞半径 25px。每 `20秒` 触发，从屏幕边缘随机生成并瞄准对侧边缘。仅有 `0.8秒` 极短预警时间，随后以高达 `1800 px/s` 的极速穿透屏幕。

### 5. 关卡与运动轨迹 (Levels & Movements)
- **Level 1 (Honest)**：静态核心，固定在屏幕左上方。
- **Level 2 (Hime)**：动态核心，以 Sine 曲线在屏幕上方左右游走。
- **Level 3 (双生核心)**：双 Boss 关卡，依据所处阶段具有复杂的协同运动逻辑：
  - **阶段 1**：每 15 秒，双 Boss 会在 2 秒内平滑互换位置。
  - **阶段 2**：分为左右半场，以 10 秒为周期，前 4 秒双 Boss 进行一上一下的镜像 Sine 曲线移动，后 6 秒待机。
  - **阶段 3**：狂暴状态。双 Boss 沿着长方形轨道的对角线进行“矩形巡逻”（8秒停驻 -> 4秒移动到下一个角），封锁玩家走位。

## 🎨 视觉反馈与打击感 (Visuals & Game Feel)

本作内置了独立计算的 `GameFeelManager`，以提供拳拳到肉的打击反馈：
1. **顿帧 (Hitstop/Freeze)**：玩家成功解析并使用攻击命中 Boss 时，物理引擎与所有逻辑将**强制暂停 `0.06秒`**，制造极强的阻滞感。
2. **屏幕震动 (Screen Shake)**：重击命中 Boss 会产生初始 12px 偏移的剧烈屏幕抖动；玩家受伤或用冲击波击杀子弹/小怪会产生 4px 的轻微抖动。抖动会随时间呈指数衰减。
3. **动态背景 (Settings - Backgrounds)**：可在设置中无缝切换 4 种底层渲染的极客风背景：
   - `0 - Cyber Grid`：幽幽滑动的青色赛博网格。
   - `1 - Starfield`：深蓝色的宇宙星空与跃迁粒子。
   - `2 - Matrix Rain`：经典的绿色黑客代码雨（动态计算字符深度）。
   - `3 - Pulse Rings`：不断向外扩散的蓝色超声波涟漪。

## 🛠️ 开发环境与构建指南 (Environment & Build)

本项目使用了现代 C++ 与最新的 **SFML 3.x** 图形库构建。

- **C++ 标准**: C++20 及以上。
- **依赖库**: SFML 3.x (由于 SFML 3 取消了 `sf::Uint8` 等自定义类型，代码中已全面重构为标准库的 `std::uint8_t`，请确保您的包含路径配置正确)。
- **资产目录 (Assets)**：请确保编译后的可执行文件同级目录下存在 `assets/` 文件夹（存放贴图）以及 `Cubic.ttf`（游戏主要字体文件）。

**构建示例 (CMake/命令行):**
```bash
# 假设已经配置好 SFML 3 的环境变量
g++ -std=c++20 main.cpp Game.cpp Boss.cpp Snake.cpp -o LastCommandMod -lsfml-graphics -lsfml-window -lsfml-system
